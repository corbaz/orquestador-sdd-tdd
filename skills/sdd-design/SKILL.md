# Skill: sdd-design

## Proposito

Definir el diseno tecnico que implementa la especificacion.

## Cuando usar

Usar con `/pi:05-design` despues de tener requisitos y escenarios claros.

## Contenido minimo

- Componentes afectados.
- Contratos publicos.
- Persistencia y migraciones si aplica.
- Estrategia de testing.
- Riesgos y tradeoffs.
- Decisiones rechazadas cuando importen.

## Regla central

No improvisar en apply. El diseno es el plano: si el plano cambia, se documenta el desvio.
