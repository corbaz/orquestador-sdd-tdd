# Skill: sdd-proposal

## Proposito

Crear una propuesta SDD breve y revisable a partir del discovery.

## Cuando usar

Usar con `/pi:03-propose` cuando el objetivo ya esta entendido y falta acordar alcance.

## Contenido minimo

- Problema.
- Objetivo.
- Alcance.
- Fuera de alcance.
- Riesgos.
- Plan de validacion.
- Proximo paso: `/pi:04-spec`.

## Criterio de calidad

La propuesta debe ser lo bastante concreta como para decidir si avanzar, pero no debe mezclar implementacion profunda. Primero se acuerda QUE y POR QUE; despues viene COMO.
